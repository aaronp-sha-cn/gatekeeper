## 2. 模块依赖关系图

```
                    core/app.py (主入口)
                         |
           +-------------+-------------+
           |             |             |
     config/          web/app.py    cli/main.py
   (settings.py)     (Flask应用)    (命令行)
           |             |
     +-----+------+      +--------+--------+
     |            |               |         |
  core/       security/      web/routes/  web/templates/
 (models,    (ids_engine,   (auth, ids,  (HTML模板)
  database,   two_factor,    health,
  scheduler)  compliance)    alerts, ...)
     |
     +-------+--------+
     |        |        |
  network/  ai_engine/  alerting/
(firewall, (traffic,    (email,
 capture)  anomaly)     webhook)
     |        |
     +---+----+
         |
     data/ (SQLite/PostgreSQL)
```

### 依赖规则

- **config/**: 零依赖，被所有模块引用
- **core/**: 仅依赖 config/，定义数据模型和基础服务
- **network/**: 依赖 config/、core/
- **ai_engine/**: 依赖 config/、core/
- **security/**: 依赖 config/、core/、network/
- **alerting/**: 依赖 config/、core/
- **web/**: 依赖 config/、core/、security/、network/、ai_engine/
- **cli/**: 依赖 config/、core/

## 3. 数据流描述

### 3.1 数据包处理流程

```
网络接口 (eth0)
      |
      v
[数据包捕获] (network/packet_capture.py)
      |  scapy 抓包
      |  IPv4 / IPv6 解析
      |
      +---> [流量分析器] (ai_engine/traffic_analyzer.py)
      |         |  特征提取
      |         |  流量统计
      |         |
      |         +---> [异常检测器] (ai_engine/anomaly_detector.py)
      |                   |  机器学习模型推理
      |                   |  异常评分
      |                   |
      |                   +---> [威胁评估]
      |                             |
      |                    +--------+--------+
      |                    |                 |
      |               [告警系统]         [IDS引擎]
      |            (alerting/)      (security/ids_engine.py)
      |               |                    |
      |          邮件/Webhook         自动阻断
      |                               防火墙规则
      |
      +---> [数据库记录] (core/models.py -> AttackLog)
```

## 7. 关键设计决策

### 单例模式
`GateKeeper` 主应用类使用线程安全的单例模式，确保全局只有一个系统实例管理所有子系统。

### 事件驱动架构
数据包捕获通过回调机制 (`register_callback`) 将数据传递给流量分析器，实现松耦合。

### 配置分层
所有配置通过环境变量 (`GK_` 前缀) 覆盖，支持开发/测试/生产环境切换。

### 安全默认值
- SECRET_KEY 未设置时自动生成随机密钥并持久化到文件
- 默认管理员密码自动生成，通过 EnvironmentFile 传递给 systemd 服务
- 首次登录强制修改密码（前端密码修改表单，实时密码强度验证）
- CSRF 保护默认启用，auth 蓝图和密码修改接口豁免（JSON API）
- flask_limiter 作为可选依赖，未安装时自动跳过速率限制
- UserRole 枚举使用大写值（SUPER_ADMIN/ADMIN/OPERATOR/VIEWER）兼容数据库存储
- 模块导入使用延迟加载模式（_lazy_import），防止单个模块失败导致系统崩溃
- 登录成功后 session.expunge(user) 防止 DetachedInstanceError
